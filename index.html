My Contribution: Backup Script

Implemented the core Bash logic for incremental snapshots.


# 1. Consistent DB Dump (Parallel)
mongodump --uri="$URI" --gzip --archive=/backup/mongo.gz

# 2. Secure Mesh Transfer (My Implementation)
# Uses --link-dest for hard-linking unchanged files
rsync -avz --delete \
  -e "ssh -o StrictHostKeyChecking=no" \
  --link-dest=../latest \
  /backup/ sdc@100.x.x.x:/snapshots/current/
✅ Key Logic: Uses `rsync` hard-links to create "Time Machine" style backups with minimal storage use.

My Contribution: Resource Governance

Applied strict cgroup limits to preventing the 900% CPU recurrence.


services:
  student-portal:
    image: sdc/portal:latest
    deploy:
      resources:
        limits:
          cpus: '0.50'     # HARD CAP: 50% of 1 core
          memory: 512M     # OOM Kill if exceeded
        reservations:
          cpus: '0.10'
          memory: 128M
Result: Even if a container is compromised, it cannot consume more than 0.5 CPU, keeping the host stable.
