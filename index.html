Hypothesis / Project Statement

"Can we engineer a self-healing, zero-trust infrastructure that automates disaster recovery and decouples stateful data from application logic to ensure 99.9% availability during security incidents?"

  • Primary Objective: Eliminate manual backup overhead.
  • Secondary Objective: Isolate media storage using S3 protocols.

Problem Statement

The "Black Swan" Event:

  • A containerized malware spiked CPU to 900%.
  • No resource governance led to host-wide instability.
  • Infrastructure was "monolithic"—images stored in code repos.
  • Manual backups took ~58 mins (unsustainable).

900%+

Resource Exhaustion

Literature Review / Market Research

  • Docker Best Practices (CIS Benchmarks): Emphasizes cpus and memory limits to prevent DoS attacks.
  • 3-2-1 Backup Rule: Industry standard requiring 3 copies of data, 2 different media, 1 off-site.
  • Stateless Architecture: Modern DevOps (12-Factor App) dictates that assets should be stored in Object Storage (S3), not local filesystems.

Research Gap / Innovation

State-of-the-Art

Manual snapshots, public-facing backup ports, local image hosting.

➡️

Our Innovation

Zero-Trust Tailscale Tunneling + Incremental Rsync + Proxied MinIO S3 Storage.

  • Difference: No public ports exposed; backups happen over a private mesh network.

System Methodology

BACKUP PIPELINE

  1. Dumps: Nightly 02:30 AM pg_dumpall & mongodump.
  2. Compression: Gzip archiving of volumes (Uptime Kuma, Vault).
  3. Transfer: rsync via Tailscale (WireGuard) to physical Bare Metal.
  4. Governance: find -mtime +7 auto-rotation.

S3 STORAGE FLOW

Client → SDC Website API → MinIO (S3 Bucket) → Isolated Volume.

Implementation of the Incremental Backup Logic


# Pushing Snapshots to Bare Metal via Secure Tunnel
if ping -c 1 -W 2 "$REMOTE_HOST" &>/dev/null; then
    log "Syncing Snapshots..."
    # Rsync over Tailscale (No SSH Port 22 exposed to internet)
    rsync -avz --delete -e "ssh -o StrictHostKeyChecking=no" \
        "$BACKUP_ROOT/" \
        "${REMOTE_USER}@${REMOTE_HOST}:${REMOTE_SNAPSHOT_PATH}/"
else
    log "ERROR: Bare Metal unreachable."
fi
