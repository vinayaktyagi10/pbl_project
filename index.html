Root Cause Analysis

A legacy deployment script allowed the creation of containers without kernel-level cgroup constraints. A compromised image executed a fork-bomb, causing the host OS to freeze and trigger a hardware-level shutdown by the cloud provider.

Automated Incremental Backups

Bash-based snapshot system using rsync hard-links 

# The Core Logic
# --link-dest creates hard links to PREVIOUS
# 0% Storage use for unchanged files.

rsync -avz --delete \
  --link-dest=../latest \
  /source/data/ \
  /backups/snapshot-$(date +%F-%H%M)/

Governance: Resource Limits

Vulnerable Config (Legacy)

services:
  student-portal:
    image: sdc/portal:latest
    # No resource constraints
    # One bug can take down the whole host

Resilient Config (Current)

services:
  student-portal:
    image: sdc/portal:latest
    deploy:
      resources:
        limits:
          cpus: '0.50'     # HARD CAP: 50% of 1 core
          memory: 512M      # OOM Kill if exceeded
        reservations:
          cpus: '0.10'
          memory: 128M

Kernel Level: Uses Linux cgroups to throttle malicious or buggy processes without crashing the OS
